## ADDED Requirements

### Requirement: Namespace prompt for config
`skey config` SHALL resolve the active namespace using the same priority as `create`: `-n` flag > `GTOOLS_SSH_NAMESPACE` env var > `"work"` default. When no `-n` flag is given, it SHALL prompt with `gum input` pre-filled with the resolved default.

#### Scenario: Namespace resolved from flag
- **WHEN** the user runs `skey config -n personal`
- **THEN** namespace SHALL be `personal` with no prompt

#### Scenario: Namespace prompt shown
- **WHEN** `skey config` is run without `-n`
- **THEN** a `gum input` prompt SHALL appear pre-filled with the resolved default

### Requirement: Per-namespace config YAML creation
`skey config` SHALL create `~/.ssh/<namespace>/.skey-config.yaml` if it does not exist. The file SHALL contain the namespace name and a placeholder `hosts` list. The target directory SHALL be created with permissions `700` if absent.

#### Scenario: Config file does not exist
- **WHEN** `~/.ssh/<namespace>/.skey-config.yaml` is absent
- **THEN** the file SHALL be created with a default YAML structure including the namespace name

#### Scenario: Config file already exists
- **WHEN** `~/.ssh/<namespace>/.skey-config.yaml` already exists
- **THEN** the file SHALL NOT be overwritten; the command SHALL proceed to apply it

### Requirement: Namespace registration in global state
After creating or confirming the per-namespace config, `skey config` SHALL register the namespace in `~/.ssh/.skey.yaml`, recording the path to its config file.

#### Scenario: New namespace registered
- **WHEN** `skey config` runs for a namespace not yet in `~/.ssh/.skey.yaml`
- **THEN** the namespace SHALL be added to the `namespaces` map in `~/.ssh/.skey.yaml`

#### Scenario: Existing namespace re-run
- **WHEN** `skey config` runs for a namespace already in `~/.ssh/.skey.yaml`
- **THEN** the state file SHALL remain unchanged (no duplicate entry)

### Requirement: Managed block in ~/.ssh/config
`skey config` SHALL upsert a managed section in `~/.ssh/config` bounded by sentinel lines. The block SHALL contain one `Include` directive per registered namespace pointing to its config YAML path.

#### Scenario: No managed block exists
- **WHEN** `~/.ssh/config` has no managed block
- **THEN** the block SHALL be appended at the end of the file

#### Scenario: Managed block already exists
- **WHEN** `~/.ssh/config` contains an existing managed block
- **THEN** the block SHALL be replaced in-place with the current state of all registered namespaces

#### Scenario: Content outside block is preserved
- **WHEN** `~/.ssh/config` has manual entries above or below the managed block
- **THEN** those entries SHALL remain untouched after the update

### Requirement: Sentinel format
The managed block SHALL be bounded by exactly these lines:
```
### generated by skey ###
```
and:
```
### end generated by skey ###
```

#### Scenario: Sentinel lines are identifiable
- **WHEN** parsing `~/.ssh/config`
- **THEN** the tool SHALL locate the block by matching these exact sentinel strings
